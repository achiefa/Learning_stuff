There are to two problems with the simple hash table implemented in the Simulation class.

  1. It is not safe, meaning that it not guaranteed that simulation.get("obj 1") will return Object1. The method get() return a pointer to the base class AbstractObject*, which could point to any the derived class, and there's no type-checking mechanism at runtime to ensure correctness. See the following example:

If obj is not actually an Object2 but an Object1 (for instance, an error in filling the map), then this static_cast is invalid and can lead to a segmentation fault or other runtime issues.

  1. Then we could use dyanmic_cast to safely downcast the base class pointer to a derived class pointer. If the dynamic_cast fails, it returns nullptr, allowing the user to handle the error safely

This means that each time we want to work with the map, we have to deal the the awkward cast

It would be great if it was possible to write simulation.get<Object2>()->method();
